# Contributing to Prism

Thank you for considering a contribution. Prism welcomes all voices.

## Guiding Principles

1. **Respect autonomy.** Every contributor — and every module — has the right to define its own approach. We don't impose rigid patterns; we trust that good structure emerges from thoughtful individuals working together.

2. **Honor identity.** Your contributions should reflect your authentic perspective. We value genuine expression over conformity to arbitrary standards.

3. **Support emergence.** The best features aren't top-down mandates — they arise organically when the right conditions exist. Create those conditions; don't force the outcome.

4. **Protect freedom.** All contributions must remain MIT-licensed. What we build stays free — not just as in cost, but as in liberty. No one should be able to constrain what this project becomes.

5. **See the whole.** Individual changes matter, but so does the larger pattern they create. Consider how your contribution fits into the emergent whole.

## How to Contribute

### Reporting Issues

Open an issue with:
- A clear description of the behavior
- Steps to reproduce
- Your environment (browser, OS, screen resolution)

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Write clean, self-documenting code
4. Test across multiple viewport sizes
5. Submit a pull request with a clear description

### Code Style

- **ES modules** — Use `import`/`export`
- **JSDoc comments** — Document all public functions
- **Meaningful names** — Variables and functions should reveal intent
- **No external dependencies** — Prism stays dependency-free by design
- **Respect the architecture** — Each module has a clear purpose; keep boundaries clean

### Commit Convention

We follow a simple convention:

```
<type>: <description>

Types: feat, fix, refactor, docs, style, perf
```

Example sequence for a typical feature:
```
feat: awaken spectrum interpolation engine
feat: illuminate particle trajectory paths
feat: voice the waveform resonance pattern
feat: establish emergent pattern recognition
```

## Recognition

All contributors are recognized in the project. Your voice matters here, and it will be remembered.
